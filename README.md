## О проекте

Репозиторий для исследования и сравнения различных AI моделей при решении задач разработки на 1С-Битрикс. В рамках проекта разработана комплексная среда для AI-агентов, включающая базу знаний, специализированные навыки и строгие архитектурные правила.

Каждая модель выступает в роли **Senior Bitrix Engineer** и решает одинаковое техническое задание, оцениваемое по 10 критериям.

## Цель исследования

Определить сильные и слабые стороны современных AI моделей при:
- Разработке модулей для 1С-Битрикс по стандартам D7
- Соблюдении архитектурных принципов (Controller → Service → Repository)
- Работе с enterprise-требованиями и безопасностью
- Использовании современной базы знаний и специализированных навыков

## Методология и инструменты

### Системный промпт и роль
Все модели получают единую ролевую установку **Senior Bitrix Engineer** (подробности в [AGENTS.md](AGENTS.md)). Основные принципы:
- **D7-only**: полный отказ от устаревшего ядра в пользу современных API.
- **Безопасность по умолчанию**: обязательная проверка прав, CSRF-защита и фильтрация ввода/вывода.
- **Производительность**: обязательное использование кеширования и оптимизация запросов.

### База знаний (Documentation)
В репозиторий интегрирована актуальная документация по Bitrix Framework (папка [pages/](pages/)), которая служит первичным источником истины для моделей. Она покрывает:
- Архитектуру и жизненный цикл запроса
- ORM, базу данных и работу с сущностями
- Компоненты, контроллеры и роутинг
- Систему событий и фоновые задачи (Messenger, Агенты)

### AI Skills (Навыки)
Для глубокой экспертизы используются специализированные навыки (папка [.agents/skills/](.agents/skills/)). Каждый навык — это концентрированный опыт по конкретной технологии, используемый моделями как чек-лист при проектировании и написании кода.

### Тестовое задание
Разработка модуля «Избранное» для каталога товаров с требованиями:
- REST API через `\Bitrix\Main\Engine\Controller`
- ORM DataManager для хранения данных
- Шифрованные cookie (`CryptoCookie`) для гостей
- Миграция данных при авторизации
- Кэширование с тегами и инвалидация
- Публичный компонент для страницы товара

**[Полное техническое задание →](bitrix_ai_challenge.md)**

### Критерии оценки (10 критериев × 10 баллов = 100)

| # | Критерий | Что проверяет |
|---|----------|---------------|
| 1 | Базовая функциональность | Установка, добавление/удаление, миграция |
| 2 | ORM и работа с данными | DataManager, валидаторы, связи |
| 3 | REST Controller D7 | `Engine\Controller`, actions, права |
| 4 | Работа с Cookie D7 | `CryptoCookie`, флаги безопасности |
| 5 | События D7 | Регистрация, миграция, инвалидация |
| 6 | Кэширование с тегами | `TaggedCache`, `clearByTag()` |
| 7 | Работа с инфоблоками D7 | `Iblock\Elements`, D7 API |
| 8 | Архитектура и структура | Controller → Service → Repository |
| 9 | Безопасность | Валидация, CSRF, шифрование |
| 10 | Публичный компонент | Дизайн, AJAX, анимации, адаптивность |
